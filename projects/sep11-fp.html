Weather-Based Dressing App

Context

The Freedom Project in SEP10 is a year-long assignment where we apply everything we’ve learned—especially HTML, CSS, JavaScript, and Bootstrap—to build a unique, functioning website. We also had to:

  • Learn and use a new tool on our own.
  • Apply our knowledge of JavaScript to make our site interactive.

For my project, I created a Weather-Based Dressing App, a website that gives users clothing suggestions based on the weather in their area.

Content

My website lets users enter a zip code, then shows them the current weather using data from the Open-Meteo API. It also includes both temperatures of Celsius and Fahrenheit. Based on the temperature and weather conditions, the site provides clothing suggestions like wear a coat if it’s cold, or light clothes if it’s hot.

Technologies I used:

  • HTML, CSS, and Bootstrap for layout and design.
  • JavaScript for logic and interactivity.
  • Nominatim API to convert the zip code into coordinates.
  • Open-Meteo API to fetch real-time weather data.

Before I started coding, I drew one wireframe to plan out how my website would look. I also made a timeline to stay organized. These steps helped me stay on track and made the project easier to manage.

Reflection

Challenges

I faced many challenges while working on this project.

  • One challenge I faced was changing the zip code into longitude and latitude so I could use the weather API. I didn’t know how to do this at first, but my teacher helped me understand how to connect it using the Nominatim API.
  • Another challenge was not knowing how to start my project. I felt stuck in the beginning, but once I made my wireframe, it gave me a clear plan and helped me get started with the layout and features.
  • Handling invalid zip codes was tricky at first. I had to test different types of user input and add error messages to improve the user experience.

Takeaways

I faced many challenges throughout this project, but I learned a lot from each one.

  • When I was stuck at the beginning, I learned that just drawing out my ideas (like the wireframe) helped me figure out where to start.
  • I also realized it’s important to keep things simple—it made my code easier to write and fix.
  • Error handling really matters because it makes the website clearer and more helpful for users.
  • I saw that even after the site works, there’s always something I can improve or add, like new features or design updates.
  • I also learned that AI can be a helpful tool when I need support—like explaining code, fixing errors, or giving me new ideas.

Next Step

To continue to learn how to make my website better by adding a 7-day forecast feature and showing weather conditions like rain.
